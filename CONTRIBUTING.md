# Contributing to Open Research Repository

Thank you for contributing. This guide is for new collaborators and early contributors.

## 1. Create a branch
- Start from `main` or the primary branch.
- Use a descriptive branch name such as `feature/add-search` or `fix/readme`.

## 2. Make focused changes
- Keep each change small and related to a single goal.
- Avoid mixing unrelated updates in the same pull request.

## 3. Write clear commits
- Use meaningful commit messages.
- Describe what changed and why.

## 4. Test before submitting
- Run any available tests locally.
- Verify that your changes work and do not break existing code.

## 5. Open a pull request
- Describe the purpose of your changes.
- Link any relevant issue or discussion.
- Keep the PR focused and easy to review.
